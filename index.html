Project Idea

Overview

The project will be a game creator where young student in rural school will collaboratively work together to turn physical creations into a digital game. Student will be required to place ‘building blocks’ of different square and rectangular sizes onto a blank board. This board will be the layout for the students to build characters and different level types. Once the student is happy with the layout of their character or level, they will then take a photo of the layout on an app. This app will gather the image and transfer it into an editable game creator where the students can assign different textures, features and animations to their character or level. Once theses features have been applied students can then test the level or game they have created being able to go back and edit any part of the game where they see fit.

Motivation

Growing up in a rural community of around 40 students in my entire school, we did not have a lot of access to new technological advancements and resources. It was not until I had grown up with these new and emerging technologies that I became fascinated with them, especially gaming. I know that if I was given the opportunity as a child (age 7-12) to have experience with this sort of technology in a rural community, I would have had started to pursue my career in I.T. then rather than waiting until I was a few years out of high school.

Project Description

The Physical
Students will have access to a blank, black board (canvas) which will be where they can physically create their characters and levels. The board will be a 20 x 20 board consisting of 1-centimetre squares. Each square will contain a small circular extrusion so that the ‘building blocks’ can be locked into them, like Lego bricks but not as tight fitting. The ‘building blocks' will also be made like Lego bricks but will be a lot flatter and can easily be placed onto the black canvas and just as easily be removed/moved around if necessary. Each square will have a 1-millimetre gap so that when a photo of the canvas is taken the app will be able to determine where the blocks will need to be placed for creation. All building blocks will be of different sizes which will vary from 1x1 all the way to 5x5 and be in a range of colours. There will be a certain number of black 1x1 block which will have a symbol on them, these will be allocated for ‘character blocks’ which will be placed when creating a level. This is to help the students physically place the character while building the level.

The Digital
Once the student/s have completed a character or level they will then take a photo using an app that that has been designed to analyse the image and determine where all the building blocks are and make a digital copy of the canvas layout. Student will have to choose whether they are creating a character or level so that they can save them in the appropriate editing format. Once the canvas has been loaded the student/s will be able to assign different textures or features to each block. The game can be as small or as large as the students want. This would accessible in the level making component of the app where students can add a new canvas ‘image’ in any direction from where they have previously created part of that level. If they wish to make a different level with new features, harder bosses, different textures, etc., they would then have to go back in the app and create a new level. The students will be able to access a particular part of the level to test it out before trying to run the full game that way they can see if there is anything that will not work. Once they have tested all part of a level, they can then run through it and when all levels have been finished, they can then test the full game. Once the game is complete and ‘bug free’ they can share it on a cloud service which will allow other users of the system to download and play the games as well. Primarily the software would be run on either an iOS or Android system on a tablet.

Team Dynamic
Students will have to work in groups of three to work on a game of their choice. They have free range to be able to make up a storyline, characters and levels so that they can code and create their game. The three different areas in which students will work in are: creating levels, creating characters and managing the coding. The idea is to let each student take turns at trying different things while collaboratively trying to create a game that not only they will enjoy but all their classmates as well.

Project Description

Tools & Technologies
The software for this app would have to be developed at a high level because the user will, essentially, need to be able to ‘code’ their own game. Although they will not be doing any hard coding, there will need to be an attention to detail about how the user interacts with the system and is able to navigate through it without any confusion and easy instructions. The hardware will need to be made so that it is precise so that the blocks can easily be placed and removed from the blank canvas board. This may require injection moulding if the game was going to be mass produced but in the preliminary stages of testing a 3D printer would suffice for physical parts.

Skills Required
Having the right people for this type of project are essential because it would have to be designed in a way that is both adult and kid friendly. One, because the kids will have to interact with the software and the physical elements of the game maker but also because the teachers will then have to try and relay all this information back to the children so that they can learn quickly and efficiently. An app designer as well as a games designer would be essential for app that needs to be created. A UI designer would be able to help with the interactively and flow of the app. For the purposes of testing stages of this project a skilled 3D printing expert would also be needed to help with the manufacture of the blank canvas board and all the building blocks.

Outcome
The outcome of this would be that schools/children who are in remote areas would be able to have access to a cheap and effective way of engaging in game development skills and give them a basic understanding of how we can use Tangible media to create something wonderful. If the first stage of this were successful, the next stage would be to increase the difficulty of ‘game making’. All the basic elements would still be the same, like putting the building blocks onto a physical canvas and transferring that to digital platform, but the difficultly of the coding might be changed. Instead of having a UI that does all of that for you students will be able to manually write in code for certain things until they are ready to create a full digital game from scratch.
